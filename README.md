## 🔍 Overview

**DrainTrack** is a comprehensive web-based drainage management system specifically designed for Muar District, Johor, Malaysia. This platform modernizes drainage infrastructure monitoring, maintenance scheduling, and flood incident reporting through an intuitive digital interface.

### 🎯 Mission
To provide efficient, data-driven drainage management solutions that help prevent flooding and maintain community safety through technology.

### 🌟 Key Highlights
- 🗺️ **Interactive Mapping** - Real-time visualization of drainage infrastructure
- 📊 **Analytics Dashboard** - Performance metrics and insights
- 📱 **Mobile Responsive** - Seamless experience across all devices
- 🔒 **Role-Based Access** - Secure multi-user system
- 📸 **Photo Documentation** - Evidence-based reporting

## 🚀 Getting Started

### 📋 Prerequisites

- 🖥️ **Web Server**: Apache (recommended)
- 🐘 **PHP**: 7.4 or higher
- 🗄️ **MySQL**: 5.7 or higher
- 📦 **Development Stack**: XAMPP/WAMP/LAMP

### ⚡ Quick Start

1. **Access the System**
   ```
   🌐 https://10.65.200.6/ai220132/index.html
   ```

2. **Choose Your Role**
   - 👑 **Admin Panel** → Full system management
   - 🛠️ **Operator Dashboard** → Task management
   - 🌐 **Public Map** → Community access

3. **Start Managing**
   - View interactive drainage maps
   - Manage maintenance tasks
   - Generate reports and analytics

## 🗺️ API Documentation

### 🔌 Core Endpoints

| Endpoint | Method | Purpose | Authentication |
|----------|--------|---------|----------------|
| `/api/drainage-points.php` | GET, POST, PUT, DELETE | Drainage point management | Required |
| `/api/drainage-lines.php` | GET, POST, PUT, DELETE | Drainage line operations | Required |
| `/api/flood-reports.php` | GET, POST | Flood incident reporting | Public/Required |
| `/api/maintenance-requests.php` | GET, POST, PUT | Maintenance task handling | Required |
| `/api/inspection-schedules.php` | GET, POST, PUT | Inspection scheduling | Required |
| `/api/task-completion-report.php` | GET, POST | Task completion tracking | Required |
| `/api/upload-image.php` | POST | Image upload handling | Required |

### 📄 Response Format
```json
{
  "status": "success|error",
  "message": "Response message",
  "data": {...},
  "timestamp": "2025-01-XX"
}
```
